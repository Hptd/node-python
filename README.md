# 简易中文节点 Python 编辑器

基于 PySide6 的图形化节点编辑器，允许用户通过拖拽和连接节点来创建可执行的数据流图。支持自定义 Python 节点函数、持久化存储和实时执行。

## 功能特性

### 1. 可视化节点编辑
- **拖拽添加节点**：从左侧节点库拖拽节点到画布
- **双击添加**：双击节点库中的节点名称快速添加
- **右键菜单添加**：在画布空白处右键（快捷键-Space），通过分类菜单或搜索快速添加节点
- **连接节点**：从输出端口拖拽到输入端口建立连接
- **框选多选**：左键拖动框选多个节点
- **视图操作**：中键平移视图，滚轮缩放

### 2. 节点库管理
- **分类管理**：节点按分类组织（输出、常量、提取、Debug、内置功能等）
- **自定义分类**：支持新建自定义分类
- **自定义节点**：通过 Python 代码动态创建新节点
- **AI 生成节点**：使用 AI 辅助生成节点代码模板，发送给第三方Ai即可生成，复制粘贴回来即可快速生成自定义节点
- **自动加载**：应用启动时自动加载自定义节点（单独文件管理）
- **右键菜单**：画布空白处右键（快捷键-Space）显示分类菜单，支持中文搜索、拼音首字母模糊搜索、英文模糊搜索快速定位节点
- **节点插件导入导出**：支持将自定义节点导出为 JSON 插件包或导入外部插件包，实现模块化功能的快速复用

### 3. 参数输入系统
- **属性面板编辑**：选中节点后在右侧面板直接输入参数值
- **输入优先级**：优先外部输入变量值，无外部输入以节点属性面板参数为准
- **智能控件匹配**：根据参数类型自动生成对应输入控件：
  - `bool` → QCheckBox（复选框）
  - `int` → QSpinBox（整数输入）
  - `float` → QDoubleSpinBox（浮点数输入）
  - `list` → JSON 格式输入框
  - `dict` → JSON 格式输入框
  - 其他 → QLineEdit（文本输入）
- **参数持久化**：参数值随节点保存到 JSON
- **默认值支持**：使用函数参数默认值作为初始值

### 4. 节点属性查看
- **文档注释**：显示节点的文档字符串
- **源代码查看**：显示节点的 Python 源代码
- **自定义节点源码**：保存并显示自定义节点的原始代码

### 5. 自定义节点
- **代码编辑器**：内置 Python 代码编辑器，无需本机配置python运行环境
- **语法验证**：自动检查代码语法
- **自动端口生成**：基于函数签名自动生成输入/输出端口
- **分类选择**：可选择或创建新分类存放自定义节点
- **插件化导出**：将自定义节点导出为 JSON 格式插件包，便于分享和复用

### 6. 主题切换
- **黑白双主题**：支持暗黑模式和明亮模式切换
- **一键切换**：点击工具栏主题按钮即可切换
- **自动保存**：主题设置自动保存，下次启动时恢复

### 7. 持久化存储
- **自定义节点存储**：自动保存到 `.node-python/custom_nodes.json`
- **图表存储**：保存为 JSON 文件，支持路径选择和命名
- **完整状态保存**：包含节点位置、连接关系、参数值
- **应用设置**：窗口状态、图形设置、执行选项等自动保存

### 8. 节点组（Group）
- **创建节点组**：选中多个节点后右键选择"创建组"（快捷键：Ctrl+G）
- **组内编辑**：双击组进入编辑模式，可独立操作组内节点
- **组合/解组**：支持将节点组合并或解散
- **组重命名**：组标题支持双击重命名

### 9. 画布内快捷智能搜索节点（快捷键-Space）
- **拼音搜索**：支持中文拼音首字母搜索（如输入"fds"匹配"浮点数"）
- **模糊匹配**：支持拼音首字母前缀、完整拼音、英文子序列匹配
- **实时过滤**：输入时实时过滤节点列表
- **优先级排序**：按匹配度排序结果
- **快速置入**：Enter 键快速置入第一个搜索结果节点

### 10. 循环节点（Loop）
- **区间循环节点（Range Loop）**：按整数范围和步长进行循环迭代
  - 输入端口：最小值、最大值、步长（均为 int 类型）
  - 输出端口：迭代值、汇总结果（所有结果列表）
  - 支持外部节点输入控制循环输入的参数
- **List 循环节点（List Loop）**：遍历列表中的每个元素
  - 输入端口：列表数据（JSON 格式的 list 类型）
  - 输出端口：迭代值、汇总结果（所有结果列表）
  - 支持外部节点输入列表数据
- **连接模式**：
  - 连接到"迭代值"：每次迭代执行的核心任务，由一个或者多个节点构成的功能链路，最后一个节点的返回值作为最终汇总结果输出的的单次结果，如果最后一个节点无返回值，则汇总结果最终端口返回为[None,None,None]
  - 连接到"汇总结果"：循环完成后执行一次
- **数据验证**：自动验证输入参数类型和格式，步长为 0 时自动修正
- **JSON 导入导出**：支持循环节点的保存和加载，自动验证数据完整性

### 11. 多线程处理节点
- **并发执行**：使用线程池并行处理列表中的每个元素
- **可配置线程数**：自定义线程数量（默认 4，自动修正无效值）
- **灵活返回顺序**：支持"按输入顺序"或"按完成顺序"返回结果
- **独立子进程执行**：每个线程在独立子进程中运行，天然线程安全
- **错误隔离**：单个线程失败不影响其他线程执行
- **实时进度显示**：控制台实时显示每个线程的完成进度
- **连接模式**：
  - 连接到"迭代值"：每次迭代执行的核心任务，由一个或者多个节点构成的功能链路，最后一个节点的返回值作为最终汇总结果输出的的单次结果，如果最后一个节点无返回值，则汇总结果最终端口返回为[None,None,None]
  - 连接到"汇总结果"：循环完成后执行一次
- **汇总结果输出**：所有线程完成后输出完整结果列表

### 11. 图表执行
- **拓扑排序**：自动确定节点执行顺序
- **批量执行**：将所有节点编译成单个脚本，只调用一次 Python 解释器
- **参数解析**：优先使用连接值，否则使用预设参数值
- **实时输出**：执行结果实时显示在控制台
- **错误处理**：捕获并显示执行错误和堆栈跟踪
- **嵌入式执行**：在独立的 `python_embedded` 环境中执行，支持第三方库

### 12. 快速复制粘贴
- **右键菜单复制**：选中节点后右键点击，选择"复制并粘贴"快速复制
- **快捷键复制**：选中节点后按 `Ctrl+D` 快速复制
- **智能连接线复制**：复制多个节点时，自动复制节点之间的连接线
- **增量位置偏移**：复制的节点自动放置在原节点右侧空白位置，避免重叠
- **支持所有节点类型**：普通节点和循环节点均支持复制
- **参数值复制**：节点的参数值、配置信息完整复制

### 13. 正则提取节点
- **智能正则生成器**：内置可视化正则表达式生成工具
- **多示例分析**：输入多个相似格式的示例，自动分析固定模式和可变模式
- **模板标记生成**：用 `[]` 标记需要提取的内容，一键生成通用正则
- **常用模板库**：提供邮箱、手机号、日期、文件名等常用正则模板
- **实时测试**：生成后可编辑并实时测试提取效果
- **语法帮助**：内置正则表达式语法快速参考


## 安装运行

### 环境要求
- Python 3.10+
- PySide6 >= 6.5.0

### 安装依赖
```bash
pip install -r requirements.txt
```

### 运行应用
```bash
python main.py
```

### 初始化嵌入式环境（可选）
首次使用或需要重置环境时：
```bash
python -m utils.setup_embedded_python install
```

或在应用中点击工具栏的 "🔧 初始化环境" 按钮。

## 使用教程

### 界面布局

软件界面分为五个主要区域：

1. **顶部工具栏**：包含运行控制、文件操作、环境管理和主题切换等功能按钮
2. **左侧节点库**：显示所有可用节点，按分类组织
3. **中央画布**：节点编辑区域，可拖拽、连接节点
4. **右侧面板**：显示选中节点的属性、参数输入、文档和源代码
5. **底部控制台**：显示执行输出和日志信息

### 工具栏按钮功能

| 按钮 | 功能说明 |
|------|----------|
| ▶ 运行 | 执行当前画布上的节点流程图 |
| ⏹ 停止 | 停止正在执行的流程 |
| 💾 保存为 JSON | 将当前图表保存为 JSON 文件 |
| 📂 加载 JSON | 从 JSON 文件加载图表（支持覆盖/增量两种模式） |
| 📤 导出节点插件 | 将自定义节点导出为 JSON 格式插件包 |
| 📥 导入节点插件 | 从 JSON 文件导入自定义节点插件包 |
| 📦 依赖管理 | 管理嵌入式 Python 环境的第三方库 |
| 🔧 初始化环境 | 初始化嵌入式 Python 环境 |
| 🌙/☀️ 切换主题 | 在暗黑模式和明亮模式之间切换 |

### 节点库操作

#### 添加节点到画布

**方法一：拖拽添加**
1. 在左侧节点库中找到需要的节点
2. 按住鼠标左键将节点拖拽到画布上
3. 松开鼠标，节点即被添加到画布

**方法二：双击添加**
1. 在节点库中双击节点名称
2. 节点会自动添加到画布中央

**方法三：右键菜单添加**
1. 在画布空白处点击右键
2. 在弹出的菜单中选择分类
3. 点击节点名称添加，或使用搜索框快速查找

#### 创建自定义分类

1. 点击左侧 "+ 新建分类" 按钮
2. 在弹出的对话框中输入分类名称
3. 点击确定，新分类会出现在节点库中

#### 创建自定义节点

1. 点击左侧 "+ 自定义节点" 按钮
2. 在代码编辑器中输入 Python 函数代码
3. 选择或创建分类
4. 点击生成节点，节点将立即出现在节点库中

**自定义节点代码规范**：
```python
def 节点名 (参数 1: 类型, 参数 2: 类型) -> 返回类型:
    """
    节点说明文档（显示在属性面板）。
    """
    return 结果
```

**规则**：
1. 必须定义且仅定义一个顶层函数
2. 函数名即为节点名
3. 参数类型决定输入控件的类型（bool/int/float/str/list/dict）
4. 返回类型注解决定是否有输出端口
5. 代码必须是合法的 Python 语法

**示例**：
```python
def 乘法 (a: int, b: int) -> int:
    """
    乘法运算节点。
    输入两个数字，返回它们的乘积。
    """
    return a * b
```

### 节点操作

#### 移动节点
- 在节点上按住左键拖动即可移动

#### 连接节点
1. 将鼠标移到源节点的输出端口（橙色圆点）
2. 按住左键拖动到目标节点的输入端口（蓝色圆点）
3. 松开鼠标，连接建立成功

#### 删除连接
- 在输入端口上双击左键，会断开该端口的所有连接

#### 删除节点
- **方法一**：选中节点后按 Delete 或者 BackSpace 键
- **方法二**：右键点击节点，选择"删除"

#### 复制并粘贴节点

**方法一：右键菜单复制**
1. 选中一个或多个节点
2. 右键点击选中的节点
3. 在弹出的菜单中选择"复制并粘贴"
4. 新节点会出现在原节点右侧的空白位置

**方法二：快捷键复制**
1. 选中一个或多个节点
2. 按 `Ctrl + D` 快捷键
3. 新节点会立即出现在原节点右侧的空白位置

**智能连接线复制**
- 如果选中的多个节点之间存在连接关系，复制时会同时复制连接线
- 如果只复制单个节点或节点之间无连接，则只复制节点

**位置偏移规则**
- 复制的节点会自动计算偏移量，放置在当前画布内容的右侧空白区域
- 保持与原节点相同的相对位置关系
- 避免与现有节点重叠

#### 多选节点

**方法一：框选多节点**
1. 在画布空白处按住鼠标左键
2. 拖动鼠标形成选择框
3. 框内的节点会被选中，可同时移动或删除

**方法二：Ctrl + 左键单击增选/取消选中**
1. 按住 Ctrl 键
2. 点击未选中的节点 → 该节点被添加到选中集合
3. 点击已选中的节点 → 该节点取消选中
4. 点击空白区域 → 保持当前选择不变

**方法三：Ctrl/Shift + 框选增选**
1. 按住 Ctrl 或 Shift 键
2. 在空白处按住左键拖动框选
3. 框内的节点会被选中，框外的节点保持原有状态
4. 支持在框选过程中实时按下/释放修饰键切换模式

### 参数输入

1. 在画布上**单击选中**一个节点
2. 在右侧 "📥 参数输入" 区域会显示该节点的所有参数
3. 根据参数类型使用对应的控件输入值：
   - 布尔值：使用复选框
   - 整数：使用数字输入框
   - 浮点数：使用小数输入框
   - 列表/字典：使用 JSON 格式文本输入
   - 其他：使用文本输入框
4. 参数值会自动保存到节点中
5. 执行时优先使用设置的值，如果端口已连接则使用连接值

### 保存和加载图表

#### 保存图表
1. 点击工具栏 "💾 保存为 JSON" 按钮
2. 在弹出的对话框中选择保存位置和文件名
3. 点击保存，图表数据（节点、连接、参数值）将被保存

#### 加载图表

**覆盖加载模式**：
1. 点击工具栏 "📂 加载 JSON" 按钮
2. 选择要加载的 JSON 文件
3. 点击打开，当前画布内容将被清空，加载新图表

**增量加载模式**：
1. 点击工具栏 "📂 加载 JSON" 按钮右侧的下拉箭头
2. 选择 "增量加载"
3. 选择要加载的 JSON 文件
4. 点击打开，保留当前画布内容，在空白位置添加新图表

### 执行流程图

1. 确保节点已正确连接
2. 设置好各节点的参数值（如需要）
3. 点击工具栏 "▶ 运行" 按钮
4. 执行结果将显示在底部控制台
5. 如需停止执行，点击 "⏹ 停止" 按钮

### 循环节点使用

#### 区间循环节点（Range Loop）

**使用步骤：**
1. 从节点库添加"区间循环"节点到画布
2. 在右侧属性面板设置循环参数：
   - 最小值：循环起始值（包含）
   - 最大值：循环结束值（不包含）
   - 步长：每次迭代的增量
3. 将"迭代值"输出端口连接到需要接收循环数据的节点
4. 将"汇总结果"输出端口连接到需要接收完整结果列表的节点
5. 运行后，节点会按照 `range(最小值，最大值，步长)` 生成迭代序列

**示例：** 最小值=0, 最大值=5, 步长=1 → 迭代值依次为：0, 1, 2, 3, 4

#### List 循环节点（List Loop）

**使用步骤：**
1. 从节点库添加"List 循环"节点到画布
2. 在右侧属性面板输入列表数据（JSON 格式，如：`["a","b","c"]`）
3. 将"迭代值"输出端口连接到需要接收循环数据的节点
4. 将"汇总结果"输出端口连接到需要接收完整结果列表的节点
5. 运行后，节点会依次输出列表中的每个元素

**示例：** 列表数据=`["apple","banana","orange"]` → 迭代值依次为：apple, banana, orange

#### 外部输入控制

循环节点支持通过外部节点输入控制循环参数：
- 区间循环：将输出整数的节点连接到"最小值"、"最大值"、"步长"端口
- List 循环：将输出列表的节点连接到"列表数据"端口

#### 数据验证

循环节点具备自动数据验证功能：
- 整数参数自动验证类型，无效时使用默认值
- 步长为 0 时自动修正为 1
- List 数据自动验证 JSON 格式，无效时使用空列表

### 数据提取节点

**数据提取节点**用于从结构化数据（字典/JSON）中提取指定路径的字段值，支持嵌套访问和数组索引。

**使用步骤：**
1. 从节点库"提取"分类中添加"数据提取"节点到画布
2. 连接数据源到 `data` 端口（或在属性面板直接输入 JSON 数据）
3. 在右侧属性面板的 `path` 参数输入提取路径

**路径语法：**

| 语法 | 说明 | 示例 |
|------|------|------|
| `字段名` | 访问字典的键 | `name` |
| `字段 1. 字段 2` | 访问嵌套字典 | `user.name` |
| `数组.索引` | 访问数组元素 | `items.0` |
| `数组 [索引]` | 访问数组元素（方括号格式） | `items[0]` |
| 组合使用 | 混合嵌套访问 | `data.items[0].name` |

**示例 1：提取嵌套字典值**
```
输入数据：
{
  "user": {
    "name": "张三",
    "age": 25
  }
}

提取路径：user.name

输出结果：张三
```

**示例 2：提取数组元素**
```
输入数据：
{
  "items": ["苹果", "香蕉", "橙子"]
}

提取路径：items.1
或：items[1]

输出结果：香蕉
```

**示例 3：复杂嵌套提取**
```
输入数据：
{
  "response": {
    "data": {
      "list": [
        {"id": 1, "title": "第一项"},
        {"id": 2, "title": "第二项"}
      ]
    }
  }
}

提取路径：response.data.list[0].title

输出结果：第一项
```

**示例 4：提取图片 URL 列表**
```
输入数据：
{
  "input": {
    "img_url": ["https://example.com/1.jpg", "https://example.com/2.jpg"]
  }
}

提取路径：input.img_url

输出结果：["https://example.com/1.jpg", "https://example.com/2.jpg"]
```

**外部输入控制**
数据提取节点支持通过外部节点输入控制参数：
- 将输出字典/JSON 的节点连接到 `data` 端口
- 将输出字符串的节点连接到 `path` 端口（动态路径）

**数据验证**
- 输入为空时返回 `None`
- 路径不存在时返回 `None`
- 自动尝试将 JSON 字符串解析为字典

**与正则提取的区别**
| 特性 | 数据提取 | 正则提取 |
|------|----------|----------|
| 输入类型 | 结构化数据（字典/JSON） | 非结构化文本 |
| 提取方式 | 路径访问 | 正则模式匹配 |
| 适用场景 | API 响应、配置文件、JSON 数据 | 日志分析、文本解析、格式提取 |
| 返回类型 | 保留原始类型 | 字符串 |

### 正则提取节点

**正则提取节点**用于从文本中提取符合正则表达式模式的内容，支持多匹配结果自动换行输出。

**使用步骤：**
1. 从节点库"提取"分类中添加"正则提取"节点到画布
2. 连接输入文本到 `input_text` 端口（或直接输入文本）

**智能生成正则表达式：**

5. 系统会自动分析所有示例，生成通用的正则表达式
6. 点击"▶ 测试提取"验证效果
7. 点击"✓ 确认使用"应用到节点

**示例 1：提取文件名前缀**

示例文本：

### 多线程处理节点

**多线程处理节点**使用线程池并发处理列表中的每个元素，适合 I/O 密集型任务或需要并行处理的场景。

**使用步骤：**
1. 从节点库"循环"分类中添加"多线程处理"节点到画布
2. 在右侧属性面板设置节点参数：
   - **输入列表**：JSON 格式的列表数据（如：`[1, 2, 3, 4, 5]`）
   - **线程数量**：并发线程数（默认 4，自动修正无效值）
   - **返回顺序**：选择"按输入顺序"或"按完成顺序"
3. 将"迭代值"输出端口连接到需要并发处理的节点
4. 将"汇总结果"输出端口连接到需要接收完整结果列表的节点
5. 运行后，节点会使用线程池并发处理列表中的每个元素

**示例 1：并发处理 URL 请求**
```
输入列表：
["https://api.example.com/1", "https://api.example.com/2", "https://api.example.com/3"]

线程数量：4
返回顺序：按输入顺序

连接节点：
迭代值 → HTTP 请求节点 → 打印节点
汇总结果 → 打印节点

输出：
[线程处理] 开始执行，输入列表长度：3，线程数：4，返回顺序：按输入顺序
[线程处理] 完成 item[0] (1/3)
[线程处理] 完成 item[1] (2/3)
[线程处理] 完成 item[2] (3/3)
[线程处理] 所有线程完成！成功：3/3
[线程处理] 汇总结果：[响应 1, 响应 2, 响应 3]
```

**示例 2：批量文件处理**
```
输入列表：
["file1.txt", "file2.txt", "file3.txt", "file4.txt"]

线程数量：2
返回顺序：按完成顺序

连接节点：
迭代值 → 文件读取节点 → 文本处理节点
汇总结果 → 数据保存节点

输出：
[线程处理] 开始执行，输入列表长度：4，线程数：2，返回顺序：按完成顺序
[线程处理] 完成 item[2] (1/4)
[线程处理] 完成 item[0] (2/4)
[线程处理] 完成 item[3] (3/4)
[线程处理] 完成 item[1] (4/4)
[线程处理] 所有线程完成！成功：4/4
[线程处理] 汇总结果：[处理结果 3, 处理结果 1, 处理结果 4, 处理结果 2]
```

**外部输入控制**
多线程处理节点支持通过外部节点输入控制参数：
- 将输出列表的节点连接到"输入列表"端口
- 将输出整数的节点连接到"线程数量"端口
- 将输出字符串的节点连接到"返回顺序"端口

**数据验证**
多线程处理节点具备自动数据验证功能：
- 输入列表为空时自动跳过执行
- 线程数量自动修正：<=0 → 1，> 列表长度 → 列表长度
- 返回顺序无效时默认使用"按输入顺序"

**返回顺序说明**
- **按输入顺序**：结果列表与输入列表顺序一致（推荐）
- **按完成顺序**：结果列表按照线程完成顺序排列（适合不关心顺序的场景）

**注意事项**
1. 每个线程在独立子进程中执行，避免线程安全问题
2. 单个线程失败不会影响其他线程的执行
3. 适合 I/O 密集型任务（如网络请求、文件读写）
4. CPU 密集型任务建议使用循环节点顺序执行
5. 线程数量不是越多越好，需根据任务类型和系统资源调整

**与循环节点的区别**
| 特性 | 循环节点 | 多线程处理节点 |
|------|----------|----------------|
| 执行方式 | 顺序执行 | 并发执行 |
| 适用场景 | 通用场景、CPU 密集型 | I/O 密集型、批量处理 |
| 执行速度 | 较慢 | 较快（并行处理） |
| 资源占用 | 低 | 较高（多线程） |
| 结果顺序 | 固定 | 可选择 |

应用设置保存在 `.node-python/settings.json`，包含：

- **窗口设置**：尺寸、位置、最大化状态
- **图形设置**：缩放速度、网格显示、对齐选项
- **节点设置**：自动保存/加载自定义节点、删除确认
- **执行设置**：自动运行、显示执行时间、错误停止
- **UI 设置**：主题、字体大小、语言、工具提示
- **嵌入式 Python**：解释器路径、自动初始化、执行环境配置
- **最近文件**：最近打开的图表文件列表
- **日志设置**：日志目录、文件名、启用状态

## 存储位置

- **设置文件**：`.node-python/settings.json`
- **自定义节点**：`.node-python/custom_nodes.json`
- **图表文件**：用户选择的 JSON 文件路径（支持循环节点）
- **日志文件**：`output_logs/output_log.txt`
